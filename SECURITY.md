# Security Policy

## Supported Versions

CertiVault is currently an architecture blueprint and has no production release.
Security fixes are applied to the latest code on the `main` branch. A versioned
support table will be added when the first release is published.

| Version | Supported |
| --- | --- |
| Latest `main` branch | Yes |
| Older commits or forks | No |

## Reporting a Vulnerability

**Do not disclose suspected vulnerabilities in public issues, discussions, pull
requests, or social media.**

Email **krishnakumarsharma8077@gmail.com** with:

- A clear description of the vulnerability and potential impact
- The affected component, branch, or commit
- Reproduction steps or a proof of concept
- Any suggested mitigation, if available
- A safe way to contact you for follow-up

You should receive an acknowledgement within 72 hours. The maintainer will
investigate, coordinate a fix where necessary, and provide status updates when
there is meaningful progress. Please allow reasonable time for remediation
before any coordinated public disclosure.

Good-faith security research that respects privacy, avoids data destruction, and
does not disrupt services is welcome. Do not access data that does not belong to
you, degrade availability, or use a vulnerability beyond what is necessary to
demonstrate it.
